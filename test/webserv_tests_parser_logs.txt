============================================================
              WEBSERV - TESTS PARSER / LOGS / TEMPS
============================================================

Contexte :
- Serveur lancé sur 127.0.0.1:8082
- Réponse actuelle attendue pour les requêtes valides :

  HTTP/1.1 200 OK
  Content-Length: 5
  Content-Type: text/plain
  Connection: close

  Hello

- Les logs serveur doivent afficher :
  date, IP, fd, méthode, path, status, taille réponse, temps de réponse.

Note :
- Pour les tests avec nc, utiliser \r\n dans les requêtes HTTP.
- Sur macOS, utiliser de préférence : nc -w 5 127.0.0.1 8082
- Le status affiché dans les logs peut encore être 0 si client.statusCode
  n'est pas encore branché dans printLog(). À corriger plus tard avec 200/400/501.
